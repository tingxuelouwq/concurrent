1. wait()与notify()/notifyAll()方法必须在同步代码块中使用

2. 当执行wait()方法时，会释放对象锁，让出CPU，进入等待状态；当执行notify()方法时，会唤醒一个处于等待该
对象锁的进程，然后继续往下执行，直到退出对象锁锁住的区域（即synchronized修饰的代码块）后才释放该对象锁。

3. wait()方法可以被中断并抛出异常

4. 监视器（Monitor)：监视器可以看做是经过特殊布置的建筑物，这个建筑物包含三个房间，分别是“特殊房间”、
“等待房间”、“大厅”。当一个消费者（Thread）进入该建筑物，它首先必须到大厅等待，调度程序将基于某些标准（
eg.FIFO）从大厅中选择一个消费者进入特殊房间；如果该消费者因为某些原因被挂起（wait），它将被调度程序安排
到等待房间，并且一段时间后会被重新分配到特殊房间。
